## 提交明细（按迭代顺序）

### 提交 1：初始版本 - 本地绝对路径加载 ANYmal B 模型

- 变更文件：主仿真代码 `.py`
- 提交说明：编写基础 MuJoCo 仿真代码，使用**本地绝对路径**加载 `anymal_b.xml`，实现模型加载 + 仿真窗口启动、基础仿真循环。
- 功能要点：
  1. 增加文件存在性校验，路径不存在时给出错误提示
  2. 调用 `mujoco.viewer` 启动被动仿真窗口
  3. 基础 `mj_step` 步进 + 画面同步逻辑

### 提交 2：路径适配优化 - 改为相对路径，兼容同级目录部署

- 变更文件：主仿真代码 `.py`
- 提交说明：替换硬编码绝对路径为**相对路径**，适配多设备运行，增加目录部署提示。
- 优化内容：
  1. 将模型路径修改为 `anybotics_anymal_b/anymal_b.xml`，脱离固定盘符依赖
  2. 补充提示文案，引导用户将模型文件夹放置在代码同级目录
  3. 保留原有文件校验与异常提示逻辑

### 提交 3：机器人防掉落优化 - 固定基座 + 物理参数调优

- 变更文件：主仿真代码 `.py`
- 提交说明：针对机器人仿真掉落、姿态漂移问题，做基座固定与物理参数优化。
- 核心改动：
  1. 锁定机器人基座 `base` 位姿，移除自由关节、绑定动捕刚体，限制刚体运动
  2. 调整仿真步长、碰撞边距、接触刚度 / 阻尼、摩擦等物理参数，提升稳定性
  3. 初始化四肢关节控制量，让机器人保持自然站立姿态
  4. 更新日志输出，标注「基座固定、不会掉落」状态

### 提交 4：模型切换 + 版权声明补充

- 变更文件：主仿真代码 `.py`、新增版权声明文本
- 提交说明：切换仿真模型为 ANYmal C，补充官方开源协议版权说明。
- 改动内容：
  1. 模型路径修改为 `anybotics_anymal_c/anymal_c.xml`，适配新机器人模型
  2. 新增 ANYbotics AG 标准 BSD 开源版权协议文本

### 提交 5：代码工程化重构 - 集中配置区 + 模块化拆分

- 变更文件：主仿真代码 `.py`
- 提交说明：代码解耦重构，抽离全局配置、封装工具函数，提升可维护性。
- 架构优化：
  1. 新增统一 `CONFIG` 配置字典，集中管理路径、位姿、仿真参数、帧率
  2. 拆分独立函数：`load_mujoco_model`（模型加载 + 异常捕获）、`configure_robot`（机器人初始化）、`run_simulation`（仿真主循环）
  3. 完善类型注解、函数注释，增加路径类型、文件合法性双重校验
  4. 统一异常捕获，细化错误输出文案

### 提交 6：核心逻辑修复 - 通过 qpos 实时锁定位姿，彻底解决坠落

- 变更文件：主仿真代码 `.py`
- 提交说明：优化基座固定逻辑，改用仿真状态量 `qpos/qvel` 实时约束位姿，根治下落、滑动问题。
- 关键修复：
  1. 不再仅修改模型静态参数，**每一帧强制重置根刚体位置、四元数姿态**
  2. 每一帧清零刚体速度 `qvel`，消除惯性漂移与下坠
  3. 简化重复配置项，统一参数调用入口

### 提交 7：参数微调 + 高度适配

- 变更文件：主仿真代码 `.py`
- 提交说明：微调机器人初始高度，避免模型穿模，复用现有稳定逻辑。
- 改动内容：上调基座初始 Z 轴高度，防止机器人与地面穿插，其余仿真、固定逻辑保持不变。

### 提交 8：功能增强 - 新增运行日志、关节状态定时打印

- 变更文件：主仿真代码 `.py`
- 提交说明：扩展仿真监控能力，增加运行时长、仿真时间、关节角度周期性输出。
- 新增功能：
  1. 配置项新增打印间隔参数，自定义日志输出频率
  2. 定时打印程序运行时长、MuJoCo 仿真时钟
  3. 截取并输出四肢关节角度数据，方便调试机器人姿态
  4. 保留帧率控制、位姿锁定全套稳定逻辑

# 附：项目整体优化总结（可选补充至文档）

1. **路径适配**：从硬编码绝对路径 → 通用相对路径，实现跨设备、跨目录运行
2. **稳定性优化**：通过固定基座、物理参数调优、逐帧约束位姿与速度，彻底解决机器人掉落、漂移、穿模问题
3. **代码架构**：面向工程化重构，模块化拆分、配置中心化、完善异常处理与类型标注，便于二次开发与参数修改
4. **调试能力**：新增运行状态、关节角度日志输出，支持仿真过程实时监控与算法调试
