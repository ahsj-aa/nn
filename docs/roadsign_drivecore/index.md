# roadsign_drivecore基于CARLA的交通标志检测与智能车辆控制系统

---

## 一、项目概述
本项目是基于CARLA 0.9.15高保真自动驾驶仿真平台开发的智能驾驶基础验证系统，采用模块化增量迭代的开发模式，构建了从环境感知、决策规划到车辆控制的完整技术闭环。项目针对课程实践与教学演示需求，重点实现了交通规则合规性驾驶、多环境适应性感知、功能安全防护等核心能力，解决了CARLA仿真环境适配、YOLOv8轻量化模型集成、实时控制延迟优化等关键技术问题。

系统采用感知-决策-控制三层经典自动驾驶架构，所有模块均采用松耦合设计，便于后续功能扩展与算法替换。通过对CARLA原生API的封装与优化，系统实现了仿真资源的自动管理与异常恢复，大幅提升了运行稳定性。同时，系统集成了YOLOv8n轻量化检测模型，在保证检测精度的前提下，将单帧推理延迟控制在50ms以内，满足实时驾驶的性能要求。

## 二、系统整体架构
本系统采用分层模块化架构设计，从上到下分为感知层、决策规划层、控制执行层和仿真支撑层四个核心层次，各层之间通过标准化接口进行数据交互，保证了系统的可扩展性和可维护性。

### 2.1 感知层
感知层是系统的"眼睛"，负责从仿真环境中获取车辆周围的环境信息，并将其转化为决策系统能够理解的结构化数据。感知层主要包含以下子模块：
- 视觉感知子模块：基于YOLOv8实现交通信号灯、限速标志、车辆、行人等目标的实时检测与分类
- 行为分析子模块：对检测到的动态目标进行运动状态分析，识别前车加速、减速、停车、变道等驾驶行为
- 车辆状态感知子模块：实时获取车辆自身的位置、速度、加速度、航向角等运动状态信息
- 环境感知子模块：获取当前仿真环境的天气、光照、时间等全局信息


### 2.2 决策规划层
决策规划层是系统的"大脑"，负责根据感知层提供的环境信息和车辆状态，做出合理的驾驶决策，并生成可执行的行驶轨迹。决策规划层主要包含以下子模块：
- 全局路径规划子模块：基于CARLA内置地图拓扑结构，生成从起点到终点的最优全局行驶路线
- 行为决策子模块：根据交通规则和实时环境信息，做出车道保持、跟车、停车、避让等高层驾驶决策
- 局部轨迹规划子模块：结合全局路径和实时障碍物信息，生成平滑、安全的局部行驶轨迹

### 2.3 控制执行层
控制执行层是系统的"手脚"，负责将决策规划层生成的轨迹转化为具体的车辆控制指令，实现车辆的精确控制。控制执行层主要包含以下子模块：
- 纵向控制子模块：采用PID算法实现车辆速度的精确调节和自适应跟车控制
- 横向控制子模块：基于纯追踪算法实现车辆的车道保持和轨迹跟踪控制
- 执行器接口子模块：将控制指令转化为CARLA仿真器能够识别的油门、刹车、转向信号

### 2.4 仿真支撑层
仿真支撑层为整个系统提供基础运行环境和辅助功能，保证系统的稳定运行和数据的有效管理。仿真支撑层主要包含以下子模块：
- 场景管理子模块：负责仿真环境的初始化、车辆生成、传感器部署和背景交通配置
- 数据采集子模块：同步采集车辆运行数据、传感器数据和环境数据，并进行结构化存储
- HMI可视化子模块：实时显示车辆状态、感知结果和系统运行信息
- 资源管理子模块：负责仿真资源的自动分配和释放，避免内存泄漏和资源占用

## 三、核心功能详细说明

### 3.1 高精度视觉感知系统
系统集成了经过专门微调的YOLOv8n轻量化检测模型，针对CARLA仿真环境中的交通目标进行了优化，在保证实时性的前提下，实现了高准确率的目标检测。
- **交通信号灯检测与识别**：支持红、黄、绿三种信号灯状态的实时检测，检测精度达到98%以上，识别延迟低于50ms。系统能够根据车辆与路口的距离，提前预判信号灯变化，在红灯亮起前平稳减速，在停止线前完全停车；绿灯亮起后自动恢复行驶；黄灯状态下根据当前车速和距离智能判断是否通过路口。
- **限速标志检测与识别**：支持多种常见限速标志的实时检测和字符识别，能够准确提取标志上的限速数值。系统会根据识别到的限速值自动调整车辆的最高行驶速度，确保车辆始终在合规速度范围内行驶。当车辆驶出限速区域后，会自动恢复到默认行驶速度。
- **动态目标检测与跟踪**：能够实时检测车辆周围的其他车辆和行人，并对其进行持续跟踪。系统会根据目标的运动状态预测其未来轨迹，为决策规划系统提供准确的环境信息。

### 3.2 智能决策与规划系统
系统实现了完整的自动驾驶决策规划能力，能够根据环境信息和交通规则，自主做出合理的驾驶决策。
- **全局路径规划**：基于CARLA内置的高精度地图，自动生成从起点到终点的最优行驶路线。系统会避开禁行区域和不可通行道路，优先选择主干道行驶。生成的路径会以可视化的方式显示在HMI界面上，便于调试和观察。
- **车道保持辅助（LKA）**：基于前视相机图像和地图信息，系统能够实时计算车辆与车道中心线的横向偏移量和航向角偏差，并通过纯追踪算法输出方向盘转角控制指令，实现车辆的车道居中保持。在直道和曲率较小的弯道上，车辆的横向偏移量能够控制在0.5米以内。
- **自适应巡航控制（ACC）**：系统能够实时检测前方车辆的速度和距离，并根据预设的安全跟车距离（默认10米）自动调整本车速度。当前车加速时，本车也会平稳加速；当前车减速时，本车会自动刹车减速；当前车完全停车时，本车也会在安全距离内平稳停车；前车起步后，本车会自动跟随起步。

### 3.3 高保真环境模拟系统
系统支持多种动态环境参数的调整，能够模拟真实世界中的不同驾驶场景，验证系统在复杂环境下的适应性。
- **多天气条件模拟**：支持晴天、多云、小雨、中雨、大雨、雾天、夜间等7种典型天气条件。用户可以通过配置文件动态调整降雨量、雾浓度、光照强度等参数。不同天气条件下，系统会自动调整相机曝光参数和检测阈值，保证感知系统的稳定性。
- **动态交通场景生成**：系统能够随机生成指定数量的背景车辆和行人，构建真实的动态交通场景。背景车辆会遵守交通规则，按照车道行驶，响应交通信号灯；行人会在人行道上行走，并在路口安全过马路。用户可以根据需要调整交通流量和行人密度。
- **时间与光照系统**：支持24小时时间流逝模拟，能够根据时间自动调整太阳位置和光照强度。系统还支持手动设置任意时间点，模拟不同时间段的光照条件。

### 3.4 功能安全与防护系统
系统内置了完善的功能安全机制，能够在发生异常情况时及时采取措施，保证车辆和人员的安全。
- **碰撞应急防护**：系统配备了虚拟碰撞传感器，能够实时检测车辆的碰撞状态。当发生碰撞时，系统会立即切断驱动扭矩，施加最大制动力，锁止车辆，防止车辆失控。同时，HMI界面会触发红色闪烁紧急告警，提示驾驶员发生碰撞。
- **异常状态处理**：系统能够实时监测自身的运行状态，当出现传感器失效、连接中断等异常情况时，会自动采取安全措施，将车辆平稳停靠在路边，并记录异常信息。
- **仿真资源自动清理**：系统实现了完善的资源管理机制，在程序退出时会自动销毁所有生成的车辆、传感器和其他仿真对象，释放占用的系统资源，避免CARLA仿真器出现内存泄漏问题。

### 3.5 数据采集与可视化系统
系统提供了全面的数据采集和可视化功能，便于用户观察系统运行状态和进行后续的数据分析。
- **全维度数据采集**：系统能够以1Hz的频率同步采集车辆的位置、速度、加速度、航向角、控制指令、交通信号灯状态、限速信息等所有核心运行数据，并将其保存为标准化的CSV格式日志文件。日志文件按时间命名，便于查找和管理。
- **HMI人机交互界面**：系统提供了简洁直观的HMI界面，实时显示车辆的当前车速、行驶模式、交通信号灯状态、剩余绿灯时间、当前天气、FPS等关键信息。感知结果会以彩色边框的形式叠加在相机图像上，不同类型的目标使用不同的颜色进行区分。
- **调试信息可视化**：系统支持显示丰富的调试信息，包括车辆行驶轨迹、预瞄点、安全距离、检测到的目标边界框等。用户可以通过配置文件开启或关闭不同的调试信息，便于算法调试和问题排查。

## 四、运行效果与演示

### 4.1 HMI界面效果
系统运行时会自动弹出HMI显示窗口，窗口左侧显示车载前视相机的实时画面，检测到的交通信号灯、车辆和行人会用不同颜色的矩形框标注出来。窗口右上角显示系统运行状态信息，包括实时车速（单位：km/h）、当前天气、交通信号灯状态、剩余绿灯时间和系统帧率（FPS）。当触发紧急制动或发生碰撞时，窗口左下角会出现红色闪烁的紧急告警提示。

### 4.2 基础驾驶效果
- **车道保持效果**：在直道行驶时，车辆能够始终保持在车道中心线附近行驶，横向偏移量小于0.3米；在曲率较小的弯道上，车辆能够平稳转向，顺利通过弯道，不会驶出车道。
- **红绿灯响应效果**：当车辆接近路口时，系统会提前检测交通信号灯状态。如果检测到红灯，车辆会在距离停止线15米左右开始平稳减速，最终完全停在停止线前；当信号灯变为绿色后，车辆会自动平稳起步，继续行驶。
- **限速管控效果**：当车辆行驶到有限速标志的路段时，系统会自动识别限速标志上的数值，并将车辆的最高行驶速度限制在该数值以内。当车辆驶出限速区域后，会自动恢复到默认的50km/h行驶速度。

### 4.3 高级功能效果
- **ACC跟车效果**：当车辆前方出现其他车辆时，系统会自动切换到跟车模式，与前车保持10米的安全距离。当前车加速时，本车也会平稳加速；当前车减速时，本车会自动刹车减速；当前车完全停车时，本车也会在安全距离内平稳停车，不会发生追尾事故。
- **碰撞防护效果**：当车辆与其他物体发生碰撞时，系统会在100ms内做出响应，立即切断油门，施加最大制动力，将车辆完全停住。同时，HMI界面会触发红色闪烁告警，提示发生碰撞事故。
- **多环境效果**：在雨天环境下，路面会出现积水和反光效果，雨刷会自动启动；在雾天环境下，能见度会降低，远处的物体变得模糊；在夜间环境下，车辆会自动开启前照灯，路面会被车灯照亮。系统在所有天气条件下都能保持稳定的检测和控制能力。

## 五、运行环境要求

### 5.1 硬件要求
- **CPU**：4核及以上处理器（推荐Intel i5-10400或AMD Ryzen 5 3600及以上），用于处理仿真逻辑和控制算法
- **GPU**：NVIDIA显卡，4GB及以上显存（推荐RTX 3060及以上），用于YOLOv8模型的GPU加速推理，提升检测帧率
- **内存**：8GB及以上（推荐16GB），用于运行CARLA仿真器和Python程序
- **存储**：10GB及以上可用SSD空间，用于存储CARLA仿真器、项目文件和数据日志，SSD能够大幅提升地图加载速度和数据写入速度

### 5.2 软件要求
- **操作系统**：Windows 10/11 64位版本，CARLA仿真器对Windows系统的兼容性最好
- **核心仿真器**：CARLA 0.9.15版本，这是目前最稳定、功能最完善的CARLA版本
- **Python运行环境**：Python 3.7.9版本，与CARLA 0.9.15版本完全兼容
- **依赖库**：
  - carla：CARLA仿真器的Python API接口
  - pygame：用于HMI界面的渲染和显示
  - numpy：用于数值计算和矩阵运算
  - ultralytics：提供YOLOv8目标检测模型支持
  - opencv-python：用于图像预处理和显示
- **虚拟环境**：推荐使用Anaconda创建独立的`carla_yolo37_env`虚拟环境运行项目，避免与其他Python项目产生依赖冲突

## 六、详细运行步骤

### 6.1 启动CARLA仿真器
1. 找到CARLA 0.9.15的安装目录，双击运行`CarlaUE4.exe`程序
2. 等待CARLA仿真器启动，这个过程可能需要1-3分钟，具体时间取决于你的电脑配置
3. 当看到城市地图完全加载出来，并且能够自由移动视角时，说明CARLA服务器已经启动成功
4. **重要提示**：必须等待CARLA仿真器完全启动并加载完所有资源后，再运行主程序，否则会出现连接失败或车辆生成失败的错误

### 6.2 激活项目虚拟环境
1. 打开Anaconda Prompt终端
2. 输入命令`conda activate carla_yolo37_env`，激活项目专用的虚拟环境
3. 确认终端提示符前面出现了`(carla_yolo37_env)`字样，说明虚拟环境激活成功

### 6.3 运行项目主程序
1. 在终端中切换到项目根目录
2. 输入命令`python Main.py`，启动项目主程序
3. 程序会自动连接到CARLA服务器，生成自动驾驶车辆，部署传感器，并加载YOLOv8模型
4. 等待几秒钟后，会自动弹出HMI显示窗口，车辆开始自动行驶，说明项目启动成功

## 八、注意事项
1. 运行项目前，必须先启动CARLA仿真器，否则会出现客户端连接失败的错误
2. 确保Python环境、依赖库版本与CARLA仿真器版本完全匹配，避免版本冲突导致程序异常
3. 每次项目运行结束，建议通过关闭HMI窗口的方式正常退出程序，这样系统会自动清理所有仿真资源，避免CARLA出现内存泄漏问题
4. 在雾天和夜间等低能见度环境下，感知系统的检测精度会略有下降，建议适当降低车辆的行驶速度
5. 长时间运行项目后，建议重启一次CARLA仿真器，释放占用的系统内存
6. 如果出现车辆不行驶、检测结果不显示等异常情况，可以尝试重启CARLA仿真器和项目程序
